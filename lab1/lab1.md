# Тестирование

Тестовый пример:

```pascal
program hello(output);

var
  x: integer;

begin
  x := 0b10010;
  writeln('x=',x);
end.
```

Вывод тестового примера на `stdout`

```
x=         18
```

# Вывод
В данной лабораторной работе была выполнена раскрутка самоприменимого компилятора на примере модельного компилятора P5: исходный текст компилятора был скопирован в отдельный файл и модифицирован так, чтобы входной язык получил новую синтаксическую возможность. В рамках задания требовалось расширить распознавание числовых литералов и обеспечить корректное преобразование их текстового представления во внутреннее значение, чтобы далее это значение могло участвовать в вычислениях и генерации кода так же, как и обычные целочисленные константы.

Затем была проведена компиляция изменённого исходного текста компилятора и проверка работоспособности расширения на небольшой программе, которая обязательно использует новую возможность языка. Тестирование подтвердило, что добавленный синтаксис двоичных констант корректно воспринимается компилятором и приводит к ожидаемому результату при выполнении программы, то есть изменения в логике обработки исходного текста действительно отражаются на поведении компилятора.

На заключительном этапе были внесены дополнительные изменения уже с использованием новой возможности языка и сохранены в следующей версии исходного текста, после чего был завершён шаг раскрутки путём компиляции обновлённого компилятора ранее полученной версией. Тем самым различия между последовательными версиями исходных файлов демонстрируют, с одной стороны, внедрение новой логики обработки входного языка, а с другой — практическое применение добавленной возможности внутри самого компилятора.
